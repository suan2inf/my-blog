# 模块06：提示词工程与编辑器集成 - 操作要求

## 项目背景
个人博客前端，使用Vue 3 + Vite技术栈。后端API已完成，现在需要集成Markdown编辑器和文章发布功能。

## 后端API信息
- 基础URL: `http://localhost:8000`
- 获取文章列表: `GET /articles?skip=&limit=&category_id=&q=`
- 获取文章详情: `GET /articles/{id}`
- 创建文章: `POST /articles` (Body: {title, content, summary, category_id})
- 更新文章: `PUT /articles/{id}`
- 获取分类列表: `GET /categories`

## 任务清单

### 任务1：安装依赖
```bash
npm install md-editor-v3
```

### 任务2：创建编辑器页面 (views/Editor.vue)

#### 功能需求：
1. **登录验证**
   - 组件挂载时检查localStorage是否有`isLoggedIn`标记
   - 未登录时显示登录弹窗（遮罩层）：
     * 用户名输入框
     * 密码输入框（type="password"）
     * 登录按钮
   - 验证信息：
     * 用户名: `admin`
     * 密码: `blog123`
   - 验证通过：存入`localStorage.setItem('isLoggedIn', 'true')`，关闭弹窗
   - 验证失败：显示错误提示"用户名或密码错误"

2. **编辑器功能**
   - 页面标题："写文章"（新建）或 "编辑文章"（编辑模式）
   - 表单区域：
     * 标题输入框（input，placeholder：请输入文章标题）
     * 分类下拉框（从API获取分类列表，选项显示name，值为id）
     * Markdown编辑器（使用md-editor-v3的MdEditor组件）
     * 操作按钮：发布/保存（主按钮样式）、取消（次要按钮样式）
   - 编辑器配置：
     * 使用`MdEditor`组件（编辑模式，不是预览模式）
     * v-model绑定文章内容
     * 导入样式：`import 'md-editor-v3/lib/style.css'`
     * 工具栏显示：标题、粗体、斜体、列表、代码块、链接
     * 最小高度500px

3. **业务逻辑**
   - 页面加载时获取分类列表填充下拉框
   - 新建模式：
     * 点击"发布"验证标题和内容不为空
     * POST /articles 创建文章
     * 成功后跳转到文章详情页 `/article/{id}`
   - 编辑模式（路由有id参数）：
     * 页面加载时GET /articles/{id}获取文章数据填充表单
     * 按钮文字改为"保存"
     * PUT /articles/{id} 更新文章
     * 成功后跳转到文章详情页
   - 点击"取消"返回首页 `/`

4. **样式要求**
   - 使用<script setup>语法
   - 原生CSS，不需要UI库
   - 简洁专业风格
   - 表单元素有适当间距
   - 登录弹窗居中显示，有遮罩层

### 任务3：升级文章详情页 (views/Article.vue)

#### 升级内容：
1. Markdown渲染
   - 使用md-editor-v3的`MdPreview`组件（预览模式）
   - 替换原来的pre标签显示文章内容
   - 导入：`import { MdPreview } from 'md-editor-v3'`

2. 代码高亮
   - 配置highlight.js
   - 安装：`npm install highlight.js`
   - 导入代码高亮样式

3. 样式
   - Markdown渲染区域添加CSS类名`.markdown-body`
   - 基础Markdown样式：
     * 标题层级样式（h1, h2, h3）
     * 段落间距
     * 代码块背景色、圆角
     * 列表样式
     * 引用块样式
   - 响应式布局

### 任务4：升级路由配置 (router/index.js)

添加路由：
```javascript
{
  path: '/editor',
  name: 'editor-new',
  component: () => import('../views/Editor.vue')
},
{
  path: '/editor/:id',
  name: 'editor-edit',
  component: () => import('../views/Editor.vue')
}
```

### 任务5：Sidebar.vue不加写文章入口

**重要**：不在侧边栏添加"写文章"按钮，保持隐藏入口设计。
只有知道/editor路径的人才能访问编辑器。
